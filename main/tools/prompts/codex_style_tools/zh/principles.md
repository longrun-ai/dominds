## 核心概念

### 1. Codex 兼容性

Codex 风格工具集设计为与 Codex provider 兼容，提供一致的工具调用体验。

**特点：**

- 工具名称与 Codex 一致
- 参数格式与 Codex 一致
- 输出格式与 Codex 一致

### 2. 只读优先

Codex 风格工具集强调只读操作，避免破坏性操作。

**工具：**

- `readonly_shell`：仅执行只读命令
- `apply_patch`：应用补丁（可审查）

### 3. 计划管理

`update_plan` 工具用于更新任务计划。

**功能：**

- 记录待办事项
- 追踪任务进度
- 更新任务状态

## 工具概览

| 工具           | 功能                |
| -------------- | ------------------- |
| apply_patch    | 应用代码补丁        |
| readonly_shell | 执行只读 Shell 命令 |
| update_plan    | 更新任务计划        |

## 最佳实践

### 1. 补丁应用

- 先审查补丁内容
- 确认无误后再应用
- 保留备份

### 2. 只读 Shell

- 仅执行查询类命令
- 避免修改系统状态

### 3. 计划更新

- 定期更新任务进度
- 保持计划与实际一致

## 限制与注意事项

1. Shell 命令仅限于只读操作
2. 补丁应用需要确认
3. 计划更新是增量式的
